# Project Report and Package Instructions

**Due date:** 5/8/2026

You will prepare a paper and a replication package for the class project. Read carefully the following instructions.

## Project paper

- Use the **ACM 2-column conference format**.
- The paper should be **well written**, **concise but detailed enough**, **easy to understand**, and **free of typos**.
- There is **no page limit**.
- Describe the **rationale** behind decisions such as data selection, approach and tool design, implementation, and evaluation methodology.
- Ensure references are **properly formatted** and **readable**.

### Important notes

- The report must contain all project details even if they were mentioned in the final presentation.
- Use feedback from the project proposal to improve the final paper quality.

## Suggested paper sections

### Abstract

Provide a general overview of:
- the problem
- motivation
- work done
- results
- implications

### Introduction

Provide:
- context of the work and problem
- problem statement from a software engineering perspective or software maintenance and evolution perspective
- motivation and beneficiaries
- summary of work done
- project contributions

### Related work

Discuss:
- attempts of prior work to solve the problem
- limitations of prior work
- how this project addresses limitations
- how this project differs from prior work

Requirements:
- include at least **3 papers**
- this section can be placed either after the introduction or after the evaluation section

### Proposed solution or tool

Include:
- **Solution description**: input, output, algorithm, data transformations, and the SE or SME data the solution incorporates
- **Justification**: technical motivation, why the solution makes sense, and how it addresses limitations of existing work
- **Tool design and implementation**: type of tool, technologies, architecture, input and output, and interaction mechanism with the user
- **Limitations**: input and output limitations, supported and unsupported systems, and supported and unsupported programming languages

### Empirical evaluation

Provide the methodology and results. Include:
- evaluation goals
- research questions
- datasets
- data statistics
- metrics
- criteria used to answer research questions
- procedure to run experiments and compute metrics
- evaluation results, including tables, graphs, and figures when helpful
- discussion of results, including interpretations, explanations, implications, and further hypotheses
- explicit answers to the research questions
- threats to validity
- examples of correct and incorrect cases produced by the solution

### Conclusions and future work

Provide:
- explicit conclusions
- lessons learned
- future research activities related to the project or other topics covered in class

### References

List all references cited in the paper. Ensure formatting is proper and consistent.
