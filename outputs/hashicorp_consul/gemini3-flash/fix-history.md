Review #1
Findings: All three `contentsRules` were silent no-ops because the YAML used the non-existent nested-list shape (`shouldOnlyContain: [- interfaces: true, ...]` / `shouldNotContainAnyOf: [- ...]`) instead of arch-go's flat-boolean keys, and the `internal/resource/**` rule additionally stacked mutually exclusive `shouldOnly*` flags; `functionsRules` used the non-existent `maxComplexity` field for `internal/**`, dropping that constraint to zero; Foundation coverage missed `types`, `version/**`, `ipaddr`, `sentinel`, `service_os`, `snapshot`, `tlsutil`, and `logging/**`; Layer 2 had no dependency rule for `proto/**` and `proto-public/**` was completely uncovered; major top-level production trees `acl/**`, `connect/**`, `envoyextensions/**`, and `troubleshoot/**` had no origin rule; the `command/** ↛ troubleshoot/**` rule was inverted (the natural direction is `command/troubleshoot/* → troubleshoot/*`); the documented intra-`agent/**` client/server split (non-server agent ↛ `agent/consul/**`) was unenforced; the YAML preamble listed `proto-public/**` as both Layer 2 and EXCLUDED; no `threshold:` block was set so the default 100% coverage gate produced `0% [FAIL]`; the Raft↔inmem pluggability ban was one-directional only; `_test.go` scoping and harness trees were undocumented; the Makefile ran `arch-go check` from a directory containing no `go.mod` and no `--config` flag; `**` globs missed root packages (`agent`, `api`, `lib`, `command`, `internal`, etc.); and the `Store` / `Repository` naming rules targeted interface names that do not exist in Consul's actual codebase (the real abstractions are `Backend` and `Reconciler`).
Fix: Rewrote all three `contentsRules` with the correct flat-boolean schema (`shouldNotContainInterfaces: true` / `shouldNotContainFunctions: true`) and added a `proto-public/**` content rule; replaced `maxComplexity: 15` with a `maxLines: 80` + `maxParameters: 6` proxy and documented the gocyclo follow-up; added Foundation `dependenciesRules` for `types`, `version`, `version/**`, `ipaddr`, `sentinel`, `service_os`, `snapshot`, `tlsutil`, `logging`, `logging/**`, `acl`, and `acl/**`; added Layer-2 dependency rules for `proto/**` and `proto-public/**` and removed `proto-public/**` from the EXCLUDED list; added origin-side rules for `connect`/`connect/**`, `envoyextensions/**`, and `troubleshoot/**`; deleted the inverted `command/** ↛ troubleshoot/**` rule (the troubleshoot rule encodes the correct direction); added 35 per-leaf rules covering the documented intra-agent client/server split (`agent/<non-server> ↛ agent/consul/**`) for ae, auto-config, blockingquery, cache, cache-types, cacheshim, checks, config, configentry, connect, debug, dns, envoyextensions, exec, grpc-external, grpc-internal, grpc-middleware, leafcert, local, log-drop, metadata, metrics, netutil, pool, proxycfg, proxycfg-glue, proxycfg-sources, router, routine-leak-checker, rpc, rpcclient, structs, structs/aclfilter, submatview, systemd, token, uiserver, and xds; replaced the impossible `internal/resource/**` `shouldOnlyContain` stack with a dependency rule banning agent/api/command imports; added the symmetric `internal/storage/inmem/** ↛ internal/storage/raft/**` ban and updated the Raft rule comment to acknowledge the symmetry; added an explicit `threshold: { compliance: 100, coverage: 0 }` block; resolved the `proto-public/**` Layer-2-vs-Excluded contradiction in the preamble; added a SCOPE NOTE documenting `_test.go` behavior and harness trees; rewrote the Makefile to use `--config "$(ARCH_CONFIG)"` and `command -v arch-go` so it can run from the consul module root while reading the YAML next to the Makefile; added directory-root mirror rules for every `**` rule so the root packages (`agent`, `api`, `lib`, `sdk`, `command`, `internal`, `connect`, `acl`) are also covered; and retargeted the naming rules from the non-existent `Store`/`Repository` interfaces to Consul's actual `Backend` (in `internal/storage/**`) and `Reconciler` (scoped to `internal/controller/**`) abstractions.

Review #2
Findings: The Makefile invoked `arch-go check --config "$(ARCH_CONFIG)"` but arch-go has no `check` subcommand and no `--config` flag (it always reads `arch-go.yml` from PWD), so CI could not actually run the rules; the Makefile also installed the stale `github.com/fdaines/arch-go@latest` v1 fork instead of the active `github.com/arch-go/arch-go/v2` org; the `connect`/`connect/**` and `envoyextensions/**` Layer-3 rules banned only `agent/consul/**` instead of the full Layer-4 `agent/**` and Layer-5 `api/**`, allowing edges like `connect/proxy → agent/structs` or `envoyextensions/extensioncommon → agent/xds/config` to silently pass; the `acl`/`acl/**` Foundation rules omitted `internal` and `internal/**` from their deny-lists (every sibling Foundation rule banned them), letting `acl/resolver → internal/resource/**` slip through; eight `agent/<leaf>/**` client/server rules (`auto-config`, `connect`, `envoyextensions`, `grpc-external`, `grpc-internal`, `grpc-middleware`, `rpcclient`, `xds`) used only the `**` glob, which under arch-go's semantics requires at least one extra path segment, so the populated leaf-root packages (e.g. `agent/xds/xds.go → agent/consul/state`) were uncovered; no production layer denied imports of test-harness trees (`testing/deployer/**`, `test-integ/**`, `testrpc`, `tools/**`, `internal/testing/**`, `internal/tools/**`) — the "Harness defense-in-depth" comment incorrectly claimed this ban was implicit; the `sdk` root rule's deny-list was shorter than `sdk/**` (missing `connect`, `connect/**`, `envoyextensions/**`, `troubleshoot/**`); three intra-`internal/**` rules (`internal/resource/**`, `internal/storage/raft/**`, `internal/storage/inmem/**`) used only the `**` form, leaving the populated leaf roots uncovered and half-enforcing the storage pluggability invariant; `grpcmocks/**` had a contents rule but no dependency rule, so a mock package could transitively pull in the agent or internal layers; `command/**` had no rule preventing direct import of `internal/**`, despite the documented expectation that the CLI consume the public Agent or API surface; the "EXCLUDED PACKAGES" preamble block still labeled `proto/private/**`, `agent/mock`, `lib/testhelpers`, and `sdk/testutil/**` as excluded when they were in fact covered by their parent globs; the Foundation preamble said "must not import a higher layer" while the rules permitted `proto/**` / `proto-public/**` imports — preamble and YAML disagreed; the connect/envoyextensions rule descriptions promised broader bans than the deny-lists actually implemented; the naming rules used the v1 simple-string `structsThatImplement: "Backend"` shape rather than v2's structured `structsThatImplement: { internal: "*Backend" }` form, and the `internal/storage` root was not in the rule's package pattern; and the `coverage: 0` threshold disabled the package-touch metric without the comment acknowledging this is a deliberate "DISABLED — re-enable as regression sentinel" choice.
Fix: Rewrote the Makefile to drop the non-existent `arch-go check --config` invocation and instead stage `arch-go.yml` into `$(PROJECT_ROOT)` and run `arch-go --verbose` from there (cleaning up the staged copy on exit), and switched the `go install` target to `github.com/arch-go/arch-go/v2@latest`; broadened the `connect`, `connect/**`, and `envoyextensions/**` deny-lists from `agent/consul/**` to the full `agent`, `agent/**`, `api`, `api/**`, `command`, `command/**` (descriptions updated to match); added `internal` and `internal/**` to both `acl` and `acl/**` deny-lists; added eight new root-mirror rules for the agent leaves whose roots were uncovered (`agent/auto-config`, `agent/connect`, `agent/envoyextensions`, `agent/grpc-external`, `agent/grpc-internal`, `agent/grpc-middleware`, `agent/rpcclient`, `agent/xds`), each banning `agent/consul` and `agent/consul/**`; replaced the placeholder "Harness defense-in-depth" comment with eleven explicit harness-deny rules covering `agent/**`, `internal/**`, `api/**`, `connect/**`, `envoyextensions/**`, `troubleshoot/**`, `lib/**`, `sdk/**`, `proto/**`, `proto-public/**`, and `acl/**`, each banning `testing/deployer/**`, `test-integ/**`, `testrpc`, `tools/**`, `internal/testing/**`, and `internal/tools/**` (with a follow-up note about the `agent/<leaf>/testutils` exception); synced the `sdk` root rule's deny-list with `sdk/**` (added `connect`, `connect/**`, `envoyextensions/**`, `troubleshoot/**`); added three new root-mirror rules for `internal/resource`, `internal/storage/raft`, and `internal/storage/inmem` mirroring their `**` counterparts; added a new `grpcmocks/**` dependency rule banning `agent`, `agent/**`, `api`, `api/**`, `command`, `command/**`, `internal`, `internal/**`, `connect`, `connect/**`, `envoyextensions/**`, and `troubleshoot/**`; added two `command`/`command/**` rules banning `internal` and `internal/**`; rewrote the "EXCLUDED PACKAGES" preamble block as a "SCOPE NOTE — Documentation-only entries" block that correctly describes which subtrees are covered by parent globs vs. truly out-of-scope; rewrote the Foundation preamble to align with the actual rule semantics (Foundation may use `proto/**` / `proto-public/**` for serialization helpers); migrated both naming rules to v2's structured form (`structsThatImplement: { internal: "*Backend" }` / `"*Reconciler"`) and added a third naming rule covering the `internal/storage` root; and rewrote the threshold comment to flag `coverage: 0` as a deliberate "DISABLED — re-enable as regression sentinel once a baseline is measured" with a recommended workflow.

Review #3
Findings: The Round-2 migration of `structsThatImplement` to the v2 structured-map form (`{ internal: "*Backend" }` / `"*Reconciler"`) caused the installed arch-go binary (a v1.5.4 fork still selected by the Makefile's `command -v arch-go` short-circuit) to abort the entire YAML unmarshal with three "cannot unmarshal !!map into string" errors at lines 1207/1216/1225, leaving every dependency, contents, naming, and function rule in the file unenforced (zero rules evaluated in CI); the three contents rules (`proto/**`, `proto-public/**`, `grpcmocks/**`) all set `shouldNotContainFunctions: true`, an overly-broad predicate that arch-go evaluates as "zero top-level functions" — false for every generated `*.pb.go` (which declares `init()`, `file_xxx_proto_init()`, gRPC `RegisterXxxServer` registration helpers as free functions) and every mockery-generated package (constructors and EXPECT helpers as free funcs), so the moment the YAML parsed every proto / mock package would hard-fail CI; the harness defense-in-depth block had no `command/**` or `command` rule, leaving the CLI — the layer most likely to drift into a "release-engineering helper imports `testing/deployer/sprawl`" pattern — completely uncovered; all 11 harness-deny rules used only the `**` form, which under arch-go's glob semantics requires at least one extra path segment, so the populated root packages `agent`, `api`, `connect`, `lib`, `acl` bypassed every harness rule (the same root-vs-`**` bug class Round 2 closed for the layer rules re-emerged in the new harness block); five harness packages that LIVE inside `agent/**` (`agent/grpc-external/testutils`, `agent/grpc-middleware/testutil`, `agent/xds/testcommon`, `agent/xds/validateupstream-test`, `agent/mock`) were not enumerated as deny targets on any rule, so production agent code could freely import the in-tree harness with no architectural feedback; `grpcmocks/**` had a layer-deny rule but no harness-deny rule, so mocks could transitively pull `testing/deployer/**` / `tools/**` into every test binary that imports them; the `internal/storage` root naming rule was structurally vacuous because the Backend interface lives at the root but its implementers live in `internal/storage/raft` and `internal/storage/inmem`; the Makefile's `cp -f`/`rm -f` of `arch-go.yml` into `$(PROJECT_ROOT)` silently overwrote any pre-existing local config without warning and left a polluted file behind on Ctrl-C / SIGTERM / OOM (no `trap`); the threshold block's `compliance: 100` gate could not detect the "YAML failed to parse" failure mode (no compliance line is ever printed in that case) and the Makefile lacked a CI assertion that arch-go produced a `Compliance:` line; the contents-rule descriptions used the phrase "complex functions" to describe `shouldNotContainFunctions: true`, misleading because the predicate is "any function" with no complexity threshold; the 35+ per-leaf intra-agent rules were strictly subsumed by a single `agent/**` glob plus an `agent` root mirror (a maintainability footgun — adding a new leaf required remembering to add two more rules, with no test that fails when forgotten); and the Foundation preamble lacked a "rule template" comment block, leaving new Foundation contributors to copy-paste from a neighboring rule and risk missing entries in the 12-target deny-list.
Fix: Reverted all three naming rules to the flat-string `structsThatImplement: "*Backend"` / `"*Reconciler"` form (accepted by both v1 and v2.x — the conservative interoperable encoding) so the YAML unmarshals against the installed binary; rewrote the Makefile `arch-check` target to (a) probe `arch-go --version` and reinstall from `github.com/arch-go/arch-go/v2@latest` whenever the on-PATH binary is not v2 (defeating the stale-v1 short-circuit), (b) refuse to start if `$(PROJECT_ROOT)/arch-go.yml` already exists (so a developer's local config cannot be silently clobbered), (c) wrap the staging-and-cleanup in a `trap '... EXIT INT TERM'` so the staged YAML is removed even on Ctrl-C / SIGTERM, (d) `tee` the arch-go output to `$(ARCH_REPORT)` and `grep -qE '^Compliance[: ]'` it, failing the build if the sentinel line is missing (catches future YAML-parse regressions); dropped `shouldNotContainFunctions: true` from the proto/proto-public/grpcmocks contents rules (kept `shouldNotContainInterfaces: true` only) and rewrote each description to explain that generated free functions are out of scope for arch-go and are enforced separately by the generator's `Code generated by ...` header; consolidated the 35+ per-leaf intra-agent client/server rules into a single `agent/**` rule plus the `agent` root mirror, leveraging arch-go's "a package never depends on itself" semantics so `agent/consul/**` as origin trivially complies; rewrote the entire harness defense-in-depth block to add (i) two new `command` / `command/**` harness rules, (ii) root-mirror harness rules for `agent`, `api`, `connect`, `lib`, `acl` (closing the root-vs-`**` glob gap), (iii) the five in-tree harness package targets (`agent/grpc-external/testutils`, `agent/grpc-middleware/testutil`, `agent/xds/testcommon`, `agent/xds/validateupstream-test`, `agent/mock`) on every layer where they could leak (`agent/**`, `agent`, `api/**`, `api`, `internal/**`, `command/**`, `command`), and (iv) a new `grpcmocks/**` harness deny rule with the standard 9-target harness payload; replaced the misleading lines-1154–1158 `agent/<leaf>/testutils` comment with a corrected explanation noting that those packages are now enumerated as deny targets on every relevant layer (and noting that `arch-go -tags=testharness` would scope future build-tag isolation if adopted); reworded the `internal/storage` root naming rule's description to label it explicitly as a "[Redundant safety net]" with a recorded explanation that the Backend interface lives at the root but implementers do not, so the rule passes vacuously today and is retained for forward compatibility; and added a "Foundation rule template" comment block to the top of `dependenciesRules` documenting the canonical 12-target deny-list, instructing future contributors to copy that template (with both a root rule and a `**` subtree rule) and explicitly noting that `proto/**` / `proto-public/**` are intentionally absent because Foundation members may import them for serialization helpers.
