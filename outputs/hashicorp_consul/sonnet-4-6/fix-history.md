Review #1
Findings: Layer 3 `agent/**` rule omitted `agent/consul/**` while matching server code; Foundation and Internal Framework package trees (ipaddr, tlsutil, sentinel, service_os, snapshot, version, internal/dnsutil, gossip, go-sso, multicluster, protohcl, protoutil, radix, resourcehcl) lacked dependency rules; `internal/resource/**` and `envoyextensions/**` were looser than the layer chart; `internal/**` Reconciler naming was over-broad; lib `contentsRules` used a no-op interface flag; `agent/consul/rate` naming text claimed `multilimiter` without a rule; test/exclude scope and transitivity limits were undocumented; FSM rule prose did not match its dependency list.
Fix: Replaced the broad Layer 3 rule with explicit non-`agent/consul` package patterns and merged Layer 5 and `agent/consul` bans into existing gRPC, cache, proxycfg, config, and related rules; added Foundation rules for tlsutil, ipaddr, sentinel, service_os, snapshot, and version; added Internal Framework dependency rules for the listed `internal/*` trees; tightened `internal/resource/**` and `envoyextensions/**`; scoped Reconciler naming to `internal/controller/**`; removed the vacuous lib interfaces `contentsRules` entry; added `RequestLimiter` naming for `agent/consul/multilimiter`; aligned FSM description with enforced paths; documented static-analysis limits, coverage semantics, and Makefile scope in comments.

Review #2
Findings: Diagram/runtime constraints (C1–C4) had no import-level mapping; harness paths (`test-integ`, `testing/deployer`, `testrpc`, `tools`, `internal/testing`, module root) lacked `package:` rules; Foundation deny lists drifted across `types`, `acl`, `proto`, `proto-public`, `logging`; `api/**`, `command/**`, `connect/**`, `acl/**`, `logging/**`, `agent/grpc-internal/**`, `agent/grpc-middleware/**`, `internal/controller/**` risked missing root packages under `/**` globs; transitive-only coupling and test-file scope were under-documented; `internal/controller` had a vacuous `contentsRules` entry; `troubleshoot/**` prose did not match the deny list; `grpcmocks` contents vs header wording was ambiguous; Makefile used non-portable `which` and lacked Windows/POSIX guidance.
Fix: Documented diagram scope, shim/transitivity limits, and test-file behavior in the YAML header and Makefile; normalized Foundation `shouldNotDependsOn` lists; added Harness `dependenciesRules` plus module-root harness bans; duplicated dependency rules for directory-root import paths (`api`, `command`, `connect`, `acl`, `logging`, `agent/grpc-internal`, `agent/grpc-middleware`, `internal/controller`); aligned `troubleshoot/**` description with enforced policy; removed the tautological `internal/controller` `contentsRules` block; clarified `grpcmocks/**` contents rule as structural-only; switched `which` to `command -v` and documented POSIX shell requirement for `make` targets.

Review #3
Findings: Runtime diagram constraints lacked import-level enforcement; `internal/tools/**` and `grpcmocks/**` had no dependency-rule subjects; directory roots `lib`, `agent/consul`, `internal/storage`, `internal/resource`, and `agent/cache` risked missing `/**` matcher coverage; transitivity/shim laundering and partial server DAG were under-documented; `command/**` banned `troubleshoot/**` conflicting with `command/troubleshoot` wrappers; proto `contentsRules` prose overstated arch-go semantics; resolver naming text referenced the wrong package; `agent/envoyextensions` vs top-level `envoyextensions` relationship was unclear; `lib/**` and `internal/storage/**` `maxReturnValues: 2` was overly strict; Makefile lacked a module-path assertion for fork layouts.
Fix: Narrowed top-level `description` to static module-prefix layering; added `dependenciesRules` for `internal/tools/**` and `grpcmocks/**`; added root `package:` mirrors for `lib`, `agent/consul`, `internal/storage`, `internal/resource`, and `agent/cache`; documented partial intra-`agent/consul` DAG and refreshed excluded-tree notes for tools/grpcmocks; removed `troubleshoot/**` from `command/**` and `command` root bans while keeping server bypass guard; aligned proto `contentsRules` descriptions with arch-go classification; fixed grpc-internal resolver naming prose; clarified `agent/envoyextensions` layering vs `envoyextensions/**`; raised `maxReturnValues` to 3 for `lib/**` and `internal/storage/**`; added `arch-verify-module` Makefile target using `go list -m`.

Review #4
Findings: Diagram/runtime topology constraints (control vs data plane, sidecar mediation, app isolation) had no static-import encoding and the disclaimer was too soft; the Layer-2 “depends on Foundation only” narrative was contradicted by rules that allow `internal/* → internal/*` peer imports; many single-directory `agent/<name>` leaves (`ae`, `auto-config`, `blockingquery`, `cacheshim`, `checks`, `configentry`, `connect`, `debug`, `dns`, `envoyextensions`, `exec`, `grpc-external`, `leafcert`, `local`, `log-drop`, `metadata`, `metrics`, `mock`, `netutil`, `pool`, `proxycfg`, `proxycfg-glue`, `proxycfg-sources`, `router`, `routine-leak-checker`, `rpc`, `rpcclient`, `structs`, `submatview`, `systemd`, `token`, `uiserver`, `xds`) were guarded only by `/**` patterns that the generator contract says match sub-packages and may not match the directory root; `connect/**` SDK rules omitted bans on `agent/**`, `internal/**`, `envoyextensions/**`, and `api/**` (asymmetric with `api/**`); transitive shim laundering was not flagged for external-lint follow-up; `agent/consul/state` interface ban could false-positive on testing-seam unexported interfaces; proto/proto-public/grpcmocks `shouldOnlyContain` rules were brittle to generator upgrades; intra-`agent/consul` DAG was only partially encoded; `agent/consul/fsm` deny list omitted `reporting` while `state` listed it; `troubleshoot/**` description claimed `api/**` was in scope while the deny list did not mention it; `lib` and `internal/storage` `maxReturnValues` rules used only `/**` subjects; `ARCH_GO_BIN` Makefile variable was captured once at parse-time and would be empty in the same invocation that installs arch-go.
Fix: Hardened the header to state diagram constraints are out of scope (CI PASS does not imply diagram conformance) and rewrote the Layer-2 description to match enforcement (“must not import Layers 3-5”) with a note that internal/** peer DAG is intentionally not encoded; added directory-root `dependenciesRules` mirrors for every single-directory Layer-3 agent leaf listed above, preserving `proxycfg`'s XDS ban and `structs`' neutral-shared-package guards; expanded `connect/**` and `connect` deny lists to include `agent/**`, `internal/**`, `envoyextensions/**`, `api/**`, `command/**`, and `troubleshoot/**` to mirror `api/**` SDK posture; added header guidance to pair these rules with `go mod graph` / depguard for shim-laundering choke points; documented the `agent/consul/state` interface-ban caveat and recommended sibling-test-package workarounds; added BRITTLENESS notes to proto, proto-public, and grpcmocks `contentsRules`; added `agent/consul/reporting` to the FSM rule and noted that the FSM deny list mirrors the state rule with cycle detection delegated to `go build`; rewrote the `troubleshoot/**` description to match the actual deny list (only `command/**` and `agent/consul/**`); added `lib` and `internal/storage` directory-root `maxReturnValues: 3` mirrors; rewrote the `arch-check` and `arch-check-verbose` Makefile recipes to re-resolve `arch-go` via `command -v` after install (with an explicit error if the binary is still missing on `$PATH`).

Review #5
Findings: All four `contentsRules` were vacuous because the YAML used arch-go's unrecognised `shouldNotContainAnyOf` / `shouldOnlyContain` nested-list shape instead of the documented flat boolean keys (`shouldNotContainInterfaces` / `shouldOnlyContainStructs`), so the state interface ban and the proto/proto-public/grpcmocks structs-only hygiene were silently dropped; the `agent/consul/state` interface ban also contradicted the `Store` naming rule, since removing the Store interface would degrade the naming check to a no-op; Layer-5 leaf isolation was broken on `troubleshoot/**` (only `command/**` and `agent/consul/**` banned) and `command/**` (only `agent/consul/**` banned), letting both leaves freely import `internal/**`, `envoyextensions/**`, and most `agent/**`; `internal/protohcl` root package had no rule (only the `/**` glob, which the generator contract says cannot match the directory root); root-mirror rules for `agent/proxycfg-sources` and `agent/rpc` matched zero packages because the paths do not exist as Go packages in the input; the `internal/controller/**` Reconciler naming rule did not cover the server-side `agent/consul/controller` sibling; the `agent/consul/multilimiter` `RequestLimiter` naming rule was vacuous because the interface lives in `agent/consul/rate`, not in `multilimiter`, so `interfaceImplementationNamingRule` resolved nothing in scope; the YAML was missing the canonical `version: 1` and `threshold:` blocks, leaving schema selection and threshold defaults implementation-defined across arch-go v1 and v2 binaries; the `internal/storage/**` framework rule (and Backend naming / `maxReturnValues` rules) matched the test-only `internal/storage/conformance` suite that the header explicitly excludes; the top-level `description:` field overstated Layer-2 enforcement (it claimed parallel sub-layers must not cross-import while the rules deliberately allow `internal/* → internal/*` peer imports); 26 of the Review #4 forward-compatibility `/**` rules for currently-leaf agent directories double-counted enforcement against their root mirrors; the `internal/multicluster → internal/controller/**` peer cycle was the most likely real Layer-2 cycle and was not banned; and the Makefile `ARCH_GO_BIN` variable remained dead code after Review #4's recipe rewrite.
Fix: Replaced all four `contentsRules` with the correct flat-boolean schema (`shouldOnlyContainStructs: true` for `proto/**`, `proto-public/**`, `grpcmocks/**`) and removed the `agent/consul/state` interface ban entirely so it no longer contradicts the Store naming rule (with a header note explaining the trade-off); rewrote the `troubleshoot/**` rule to enforce strict-leaf isolation against `agent/**`, `internal/**`, `envoyextensions/**`, `command/**`, and `api/**`, and tightened both `command/**` and `command` root rules to also ban `internal/**` and `envoyextensions/**` (keeping `agent/**` allowed for legitimate CLI entrypoints); added a directory-root mirror rule for `internal/protohcl`; deleted the vacuous `agent/proxycfg-sources` and `agent/rpc` root mirrors (replaced with comment markers explaining why); added a parallel Reconciler naming rule scoped to `agent/consul/controller`; switched the `agent/consul/multilimiter` naming rule to use the `*Limiter` wildcard pattern instead of a same-package interface lookup; added `version: 1` and an explicit `threshold: { compliance: 100, coverage: 0 }` block at the top of the YAML; split the `internal/storage/**` dependency rule into per-backend rules for `internal/storage/inmem` and `internal/storage/raft` plus a relaxed rule for `internal/storage/conformance` (banning only `command/**` and `troubleshoot/**`); rewrote the top-level `description:` to accurately advertise the asymmetric Layer-5 leaf isolation, the deliberate Layer-2 peer-import allowance with the single multicluster exception, and the diagram-out-of-scope posture; added a FORWARD-COMPAT NOTE in the Layer-3 root-mirror section comment explaining that 26 paired `/**` rules are scaffolding for currently-leaf directories; added `internal/controller/**` to the `internal/multicluster` deny list as the single targeted Layer-2 peer ban; deleted the dead `ARCH_GO_BIN` variable from the Makefile (replaced with a clarifying comment) and added a new `arch-version-check` target that asserts the installed `arch-go` binary is the v1 fork (`fdaines/arch-go`) the YAML targets.
